## 🏆 推薦方案

### **第一選擇：Dessert Passport（護照）🥇**

**理由：**
1. **已有日常習慣** - 護照集章天生需要「每日回訪」
2. **完美契合 O2O** - 線上簽到 + 到店集章 + 線上確認
3. **與運籤無衝突** - Fortune Slip 是「樂趣」，簽到是「習慣」
4. **積分+獎勵系統** - 簽到天數 → 解鎖印章 → 兌換獎品
5. **設計簡潔** - LocalStorage 可輕鬆支持日期追蹤

**實施方案：**
```
Passport 護照首頁
├─ 現有：運籤卡片
├─ 新增：簽到卡片
│  ├─ 顯示連簽天數
│  ├─ 顯示今日是否簽到
│  ├─ 一鍵簽到按鈕
│  └─ 簽到日曆
├─ 運籤加成：簽到後運籤 x1.5 倍幸運
└─ 連簽獎勵：簽滿 7/30/100 天 → 解鎖特殊印章
```

**技術成本：** 低 ✅
- 護照已用 LocalStorage
- 只需添加日期檢查邏輯
- 無需後端

---

### **第二選擇：Gacha 扭蛋 🥈**

**理由：**
1. **天生遊戲感** - 扭蛋本身就有上癮性
2. **日常機制已有** - Fortune Slip 每日可轉
3. **複合體驗** - 簽到 → 運籤 → 扭蛋奇蹟
4. **微課金友好** - 簽到 → 累積虛擬幣 → 免費扭蛋

**實施方案：**
```
Gacha 首頁
├─ 簽到條紋進度（視覺化）
├─ 「今日簽到」大按鈕
├─ 累積簽到天數顯示
└─ 連簽達成時：
   ├─ 解鎖每日免費扭蛋次數
   ├─ 運籤倍率提升
   └─ 隱藏彩蛋激活
```

**技術成本：** 低 ✅
- 已有 Vite React 前端
- LocalStorage 支持

---

### **第三選擇：Moon Map 導覽 🥉**

**理由：**
1. **與情緒 Check-in 結合** - 「你今天是什麼狀態？」＋「簽到」
2. **品牌策展入口** - 簽到累積進度條，解鎖限定內容
3. **多元進度** - 不只消費，還有「每日心情」參與

**實施方案：**
```
Moon Map 首頁
├─ 「今日心情簽到」卡片
│  ├─ 情緒狀態選擇（開心/平靜/迷茫等）
│  ├─ 簽到確認
│  └─ 累積進度
├─ 心情軌跡視覺化
│  ├─ 周心情圓圈圖
│  └─ 月心情曲線
└─ 簽到獎勵：
   ├─ 情緒一致度達成 → 推薦指定甜點
   └─ 簽到連續 → 解鎖 Hidden Menu
```

**技術成本：** 中 ⚠️
- 需要情緒數據儲存
- 需要推薦邏輯

---

### **不推薦：MBTI Lab 和 Dessert Booking**

**MBTI Lab 原因：**
- 測驗是「深度一次性體驗」，不適合日常簽到
- 會沖淡測驗的專注度
- 複測應該是自主選擇，不是被迫簽到

**Dessert Booking 原因：**
- 訂購系統應該專注交易，不應混入遊戲化
- 日活低（只有購買時進入）
- 添加簽到會破壞購物流程
