# 品項照片命名與更新指南

**共用圖床**：本專案（Dessert-Booking）與 **moon_map_original** 使用**同一個 Supabase 圖床**。產品照片上傳、命名與資料庫更新流程請以 **moon_map_original** 為準，完成後本專案會自動顯示相同圖片。詳見同層目錄的 **moon_map_original** 專案內：`docs/產品照片_Supabase上傳與同步.md`；本專案摘要見 `docs/共用圖床說明.md`。

以下為本專案單獨匯出品項、取名邏輯與規格參考；實際上傳與 SQL 更新請依 moon_map_original 的腳本與文件操作。

---

## 一、取得目前所有品項清單（與網站顯示一致）

### 方式 A：從網站 API 匯出（推薦）

資料來源與前台／後台顯示相同。

1. **本機**：先執行 `npm run dev`，再執行：
   ```bash
   npm run export:menu
   ```
2. **線上**：直接執行（把網址換成你的部署網址）：
   ```bash
   BASE_URL=https://你的網站網址 npm run export:menu
   ```
3. 產生的 CSV 會寫入 **`docs/品項清單_照片更新用.csv`**，可直接用試算表開啟編輯。

### 方式 B：從 Supabase SQL 匯出

1. 開啟 **Supabase Dashboard** → 專案 → **SQL Editor**
2. 新增查詢，貼上並執行：**`scripts/EXPORT_MENU_ITEMS.sql`**
3. 將查詢結果複製到試算表或本文件下方「品項清單」表格，方便對照更新

## 二、取名邏輯（方便網站應用）

### 規則

| 規則 | 說明 | 範例 |
|------|------|------|
| **格式** | `{分類英文}-{品名英文}.jpg` | `cake-strawberry-tart.jpg` |
| **僅用** | 小寫英文、數字、連字號 `-` | 不要空格、中文、特殊符號 |
| **分類代碼** | 用簡短英文代表分類 | cake, drink, dessert, seasonal |

### 分類代碼建議

| 分類（中文） | 建議代碼 | 說明 |
|--------------|----------|------|
| 蛋糕／塔類 | cake | 蛋糕、塔、派 |
| 飲品 | drink | 咖啡、茶、飲料 |
| 甜點／其他 | dessert | 餅乾、小點、其他 |
| 季節限定 | seasonal | 節慶、限定款 |

### 品名轉英文檔名

- 空格 → 連字號 `-`
- 只保留英文字母、數字、連字號，其餘刪除
- 全小寫

**範例：**

| 品名     | 建議檔名              |
|----------|-----------------------|
| 草莓塔   | cake-strawberry-tart   |
| 靜默的流動 | drink-silent-flow    |
| 本季限定 巧克力 | seasonal-chocolate   |

### 圖床與網址（與 moon_map_original 共用）

- **圖床**：Supabase Storage，bucket `moon-island-assets`，路徑 `menu-items/`
- **公開 URL 範例**：`https://你的專案.supabase.co/storage/v1/object/public/moon-island-assets/menu-items/classic_tiramisu.webp`
- **檔名**：請使用 **moon_map_original** 的「英文代號 (Image ID)」，例如 `baileys_tiramisu.webp`、`classic_tiramisu.webp`，對照表見該專案或桌面的 `產品品項與檔案名稱對照表.csv`。

網站會直接使用 `menu_items.image_url` 或 `menu_items.image`（同一筆資料），上傳與更新請在 moon_map_original 完成，本專案會自動同步顯示。

## 三、照片規格建議

| 項目     | 建議 |
|----------|------|
| **尺寸** | **800 × 800 px**（正方形，與商品卡一致） |
| **比例** | 1:1，避免變形 |
| **格式** | JPG 或 WebP，檔案不要過大（單張 < 500KB 佳） |

---

## 三之一、上傳到 Supabase 的建議（尺寸與檔案類型）

若圖片是放到 **Supabase Storage**（或貼進資料表前的圖床），建議依用途分開規格：

### 商品圖（menu_items，首頁／購物車用）

| 項目 | 建議 | 說明 |
|------|------|------|
| **尺寸** | **800 × 800 px**（或 1024 × 1024） | 網站用 `aspect-square` 1:1 顯示，800 在手機與桌機都夠清晰 |
| **比例** | **1:1** | 避免被裁切或變形 |
| **檔案類型** | **WebP** 或 **JPG** | WebP 較小、畫質佳；JPG 相容性最好。避免 PNG（檔案大）除非需透明 |
| **檔案大小** | **單張 &lt; 500KB**（理想 &lt; 300KB） | 載入快、Supabase 流量也較省 |

### Banner 圖（banners，推廣區塊用）

| 項目 | 建議 | 說明 |
|------|------|------|
| **尺寸** | **800 × 800 px** 或 **600 × 600 px** | 目前版面顯示為小圖（約 96×96px），不需過大 |
| **比例** | **1:1** 或 **橫幅 2:1**（如 800×400） | 依未來是否改版放大而定 |
| **檔案類型** | **WebP** 或 **JPG** | 同上 |
| **檔案大小** | **單張 &lt; 300KB** | 足夠即可 |

### 通用建議

- **格式優先**：**WebP**（體積小、畫質好）＞ **JPG**（相容性最好）＞ PNG（僅在需要透明時用）。
- **Supabase Storage**：單檔預設上限約 50MB，一般商品圖 / Banner 用上述規格即可；上傳後取得**公開 URL** 貼到 `menu_items.image_url` 或 `banners.image_url`。
- **檔名**：用英文、小寫、連字號（如 `cake-strawberry-tart.webp`），避免空格與特殊字元，方便管理與 CDN 快取。

## 五、更新照片到 Supabase

### 方式 A：Supabase Dashboard

1. **Table Editor** → 選 **menu_items**
2. 找到該筆品項 → 編輯 **image_url** 欄位
3. 貼上 Cloudinary（或圖床）的完整圖片網址

### 方式 B：SQL 批次更新

```sql
-- 單筆更新範例
UPDATE menu_items
SET image_url = 'https://res.cloudinary.com/xxx/image/upload/.../cake-strawberry-tart.jpg'
WHERE id = '該品項的 id';
```

### 方式 C：後台管理（若有建）

若專案有 Admin 商品編輯頁，從後台上傳或貼網址更新即可。

## 六、檢查網站是否正確顯示

更新後到前台「本季精選」或各分類確認：

- 圖片是否 1:1、不被壓扁
- 載入失敗時是否出現預設圖示

若圖片網址錯誤或失效，畫面上會顯示預設的 🍰 圖示。
