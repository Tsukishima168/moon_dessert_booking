# 本專案：商品分類與取名架構

## 一、資料來源（後台／前端一致）

- **分類**：Supabase 表 `menu_categories`
- **商品**：Supabase 表 `menu_items` + `menu_variants`
- **前端**：`/api/categories`、`/api/menu` 回傳的資料即為畫面上顯示的內容

---

## 二、商品分類架構

### 1. 表：`menu_categories`

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | 字串／數字 | 分類唯一識別，**與 menu_items.category 對應** |
| name | 字串 | 分類名稱（畫面上顯示，如「蛋糕」「飲品」） |
| title | 字串 | 若無 name 時程式會用 title |
| sort_order | 數字 | 排序用，數字越小越前面 |

- 前端會依 `sort_order` 排序分類，再依 `category.id` 分組商品。
- **重要**：`menu_items.category` 填的值必須等於某個 `menu_categories.id`，商品才會出現在該分類底下。

### 2. 分類代碼對照（照片／檔名用）

| 分類代碼 (code) | 分類名稱 (中文) | Category (English) | 說明 |
|-----------------|-----------------|--------------------|------|
| cake | 蛋糕／塔類 | Cake / Tart | 蛋糕、塔、派 |
| drink | 飲品 | Drink | 咖啡、茶、飲料 |
| dessert | 甜點／其他 | Dessert / Other | 餅乾、小點、其他 |
| seasonal | 季節限定 | Seasonal | 節慶、限定款 |

- 分類代碼用於：**照片檔名**、**匯出 CSV 的「分類代碼」欄位**，方便辨識與整理。
- 實際畫面上的分類以 `menu_categories` 的 `name` 為準，可與上表對應使用。

### 3. 飲品類特別邏輯

- 前端會把「名稱含 drink／飲品／飲料」的分類視為**僅門市供應、不開放線上預訂**。
- 這類商品會顯示「僅供門市內用」、不顯示加入購物車，且不會出現在「本季精選」可預訂清單。

## 三、商品取名與欄位架構

### 1. 表：`menu_items`

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | 字串／數字 | 商品唯一識別 |
| name | 字串 | **品名**（畫面上顯示；若無則用 title） |
| title | 字串 | 品名備用欄位 |
| description | 字串 | 商品說明 |
| category | 字串 | **分類**，需等於某個 `menu_categories.id` |
| category_id | 字串 | 若用 FK 時程式會轉成字串當 category 用 |
| image_url | 字串 | 商品圖片網址（畫面上顯示） |
| image | 字串 | 圖片網址備用欄位 |
| is_available | 布林 | 是否上架（false 時前端顯示為已售完） |

### 2. 表：`menu_variants`

| 欄位 | 型別 | 說明 |
|------|------|------|
| id | 字串／數字 | 規格唯一識別 |
| menu_item_id | 字串／數字 | 所屬商品 id |
| variant_name | 字串 | **規格名稱**（如「6 吋」「單片」） |
| spec | 字串 | 規格備用欄位 |
| price | 數字 | 該規格價格（程式會自動去掉 $、逗號等） |

- 一個商品可有多筆 variant，前端會依規格顯示不同價格與選項。

### 3. 取名／顯示邏輯整理

| 用途 | 來源 | 說明 |
|------|------|------|
| 畫面上「分類名稱」 | `menu_categories.name` | 依 sort_order 排序後顯示 |
| 畫面上「品名」 | `menu_items.name` 或 `title` | 商品主標題 |
| 畫面上「規格」 | `menu_variants.variant_name` 或 `spec` | 每個價位的選項名稱 |
| 商品所屬分類 | `menu_items.category` = `menu_categories.id` | 用來分組、篩選 |
| 照片檔名建議 | `{分類代碼}-{品名英文}.jpg` | 例：cake-strawberry-tart.jpg |

---

## 四、照片與檔案命名架構

- **格式**：`{分類代碼}-{品名英文}.jpg`，小寫、連字號、僅英文／數字。
- **分類代碼**：見上方「分類代碼對照」。
- **品名轉英文**：空格改 `-`，只留英文、數字、連字號，全小寫。
- **範例**：品名「靜默的流動」、分類飲品 → 建議檔名 `drink-silent-flow.jpg`。
- 完整對照與規格見：`docs/品項照片命名與更新指南.md`、`docs/分類對照表.csv`。
