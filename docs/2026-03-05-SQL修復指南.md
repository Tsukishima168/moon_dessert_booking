## 步驟 1: 修復 orders 表缺失的列

### 執行位置
**Supabase Dashboard** → **SQL Editor** → 複製貼上以下代碼 → **Execute**

```sql
-- 添加缺失的 utm 追蹤列
ALTER TABLE orders
ADD COLUMN IF NOT EXISTS source_from VARCHAR(50),
ADD COLUMN IF NOT EXISTS utm_source VARCHAR(100),
ADD COLUMN IF NOT EXISTS utm_medium VARCHAR(100),
ADD COLUMN IF NOT EXISTS utm_campaign VARCHAR(100),
ADD COLUMN IF NOT EXISTS utm_content VARCHAR(100),
ADD COLUMN IF NOT EXISTS user_id UUID REFERENCES auth.users(id) ON DELETE SET NULL;

-- 添加索引以提高查詢性能
CREATE INDEX IF NOT EXISTS idx_orders_utm_source ON orders(utm_source);
CREATE INDEX IF NOT EXISTS idx_orders_user_id ON orders(user_id);

-- 驗證
SELECT column_name, data_type 
FROM information_schema.columns 
WHERE table_name = 'orders' 
ORDER BY ordinal_position;
```

✅ 執行成功，0 errors
- 6 列新增成功（source_from, utm_source, utm_medium, utm_campaign, utm_content, user_id）
- 2 個索引建立成功

## 步驟 2: 修復時區轉換問題

### 執行位置
**Supabase Dashboard** → **SQL Editor** → 複製貼上以下代碼 → **Execute**

```sql
-- 刪除舊的 RPC 函數
DROP FUNCTION IF EXISTS check_daily_capacity(DATE, VARCHAR);
DROP FUNCTION IF EXISTS check_daily_capacity(DATE, VARCHAR) CASCADE;

-- 重新創建 RPC 函數，修復時區轉換
CREATE OR REPLACE FUNCTION check_daily_capacity(
    check_date DATE,
    delivery_method_param VARCHAR
)
RETURNS TABLE(
    date_str VARCHAR,
    total_capacity INTEGER,
    booked_count INTEGER,
    available_count INTEGER,
    is_available BOOLEAN
) AS $$
BEGIN
    RETURN QUERY
    SELECT 
        check_date::VARCHAR,
        COALESCE(bs.max_daily_orders, 50)::INTEGER,
        COALESCE(COUNT(o.id)::INTEGER, 0),
        COALESCE(bs.max_daily_orders, 50) - COALESCE(COUNT(o.id)::INTEGER, 0),
        CASE 
            WHEN COALESCE(bs.max_daily_orders, 50) - COALESCE(COUNT(o.id)::INTEGER, 0) > 0 THEN TRUE
            ELSE FALSE
        END
    FROM business_settings bs
    LEFT JOIN orders o ON 
        DATE(o.pickup_time AT TIME ZONE 'Asia/Taipei') = check_date
        AND o.delivery_method = delivery_method_param
        AND o.is_cancelled = FALSE
    GROUP BY bs.max_daily_orders;
END;
$$ LANGUAGE plpgsql STABLE;

-- 驗證函數存在
SELECT routine_name 
FROM information_schema.routines 
WHERE routine_name = 'check_daily_capacity';
```

### 預期結果
✅ 執行成功，0 errors
- 舊函數已刪除
- 新函數已建立（修復了時區轉換：`AT TIME ZONE 'Asia/Taipei'`）
- SELECT 查詢返回函數名稱

## 若執行有誤

### 錯誤: "function check_daily_capacity already exists"
**解決**: 使用 `DROP FUNCTION ... CASCADE;` 刪除

### 錯誤: "column source_from already exists"  
**解決**: 使用 `ADD COLUMN IF NOT EXISTS` (已內含)

### 錯誤: 時區轉換仍然失敗
**檢查**: 
1. 確認 `pickup_time` 列是否為 TEXT 類型
2. 確認時區格式 `'Asia/Taipei'` 是否正確
3. 檢查 PostgreSQL 版本支持
