## 🎯 點數體系設計

### 三種點數類型

```
┌─ 通用點數 (Universal Points)
│  ├─ 來源：各專案都能產生
│  ├─ 比例：1 通用點 = 1 元購物金
│  ├─ 可兌換：所有商品/服務
│  └─ 例如：完成測驗 +10 通用點
│
├─ 專項點數 (Category Points)
│  ├─ 洞察點 (Insight Points) - MBTI 專用
│  ├─ 遊戲點 (Game Points) - 遊戲專用
│  ├─ 甜點點 (Sweet Points) - 美食專用
│  ├─ 用途：專項排行榜、專項成就
│  └─ 兌換比例：5 專項點 = 1 通用點
│
└─ 等級點數 (Level Progress)
   ├─ 自動計算：基於通用點數
   ├─ 用途：升級分級（銀卡→金卡→鑽石卡）
   └─ 消費點 + 活動點 = 等級進度
```

## 💳 等級系統 - 統一計算

### 等級判定公式

```
基礎分數 = 通用點數 (忽略專項點)

銀卡 (Silver)
├─ 條件：首次加入 or 年通用點 > 100
├─ 特權：基礎積分倍數 1x、生日禮
└─ 保留期：永久

金卡 (Gold)
├─ 條件：年通用點 ≥ 2000
│        或達成「三星達成」(任意三項活動各達 1000+ 點)
├─ 特權：積分倍數 1.2x、免運、季度優惠
└─ 保留期：滾動計算（當年度）

鑽石卡 (Platinum)
├─ 條件：年通用點 ≥ 6000
│        或達成「全能達人」(至少 3 個不同專項各達 500+ 點)
├─ 特權：積分倍數 1.5x、全年免運、月度禮物
└─ 保留期：滾動計算（當年度）

皇冠卡 (Crown)
├─ 條件：年通用點 ≥ 12000
│        或達成「宇宙探險家」(4 個專項各達 1000+ 點)
│        或邀請 10 位新用戶 (各消費 1 次以上)
├─ 特權：積分倍數 2x、終身免運、月度 $500 購物金
└─ 保留期：終身 (除非年度無消費)
```

### 等級進度條

```
進度 = min(年通用點 / 等級上限, 100%)

例如：
目標: 達到金卡 (需 2000 點)
當前: 600 點
進度: 600 / 2000 = 30% ████░░░░░░

目標: 達到鑽石卡 (需 6000 點)  
當前: 4800 點
進度: 4800 / 6000 = 80% ████████░░
```

## 💱 兌換商城 - 統一中心 (Passport)

### 兌換比例

```
通用點數兌換
├─ 100 點 = NT$20 購物金 (所有專案通用)
├─ 250 點 = NT$60 購物金
├─ 500 點 = NT$120 購物金 + 精美禮物
└─ 1000 點 = NT$250 購物金 + VIP 通行證 (1個月)

專項點數兌換
├─ 洞察點
│  └─ 500 洞察點 = $50 購物金 (MBTI 測驗折扣)
│
├─ 遊戲點
│  └─ 500 遊戲點 = 遊戲內寶物包
│
└─ 甜點點
   └─ 500 甜點點 = NT$50 甜點購物金
```

### 兌換流程

```
會員進入 Passport 中心
  ↓
選擇兌換項目
  ├─ 購物金 (使用通用點)
  ├─ 專項獎勵 (使用專項點)
  └─ 禮物/體驗
  ↓
點擊「兌換」
  ↓
通用點 -100 | 專項點 -500 (自動扣除)
  ↓
獲得兌換碼/購物金 (自動發送)
  ↓
使用 (在各專案系統中)
```

## 🔄 數據同步架構

### 各專案存儲結構

```
MBTI 專案 DB
└─ users 表
   ├─ user_id
   ├─ username
   ├─ mbti_type
   └─ passport_id ← 外鍵連接到 Passport

Dessert-Booking DB
└─ members 表
   ├─ member_id
   ├─ name, phone
   ├─ level
   └─ passport_id ← 外鍵連接到 Passport

遊戲系統 DB
└─ players 表
   ├─ player_id
   ├─ username
   ├─ game_score
   └─ passport_id ← 外鍵連接到 Passport

Passport 中心 DB (單一真相來源)
├─ passports 表
│  ├─ passport_id (主鍵)
│  ├─ user_email (統一識別)
│  ├─ universal_points ✨
│  ├─ level
│  ├─ created_at, updated_at
│  └─ linked_accounts (JSONB) - 連接的各專案帳號
│
├─ point_sources 表
│  ├─ passport_id
│  ├─ source_project (mbti|game|dessert|...)
│  ├─ activity_type
│  ├─ points_earned
│  ├─ created_at
│  └─ metadata (JSONB)
│
├─ category_points 表
│  ├─ passport_id
│  ├─ category (insight|game|sweet)
│  ├─ balance
│  └─ updated_at
│
└─ member_badges 表
   ├─ passport_id
   ├─ badge_name
   ├─ unlocked_at
   └─ category
```

### 同步流程 (N8N 或 API)

```
專案發生積分事件
  ↓
發送 Webhook 到 Passport 中心
  ├─ Webhook 格式：
  │  {
  │    "source": "dessert-booking",
  │    "user_email": "user@example.com",
  │    "activity": "purchase",
  │    "points": 100,
  │    "category": "sweet",
  │    "timestamp": "2026-02-19T10:00:00Z"
  │  }
  │
  ↓
Passport 驗證並更新
  ├─ 檢查 user_email 是否存在
  ├─ 新增積分交易記錄
  ├─ 更新 universal_points 和 category_points
  ├─ 檢查徽章解鎖條件
  ├─ 檢查等級晉升條件
  └─ 返回 {success: true, new_balance: 600}
  ↓
各專案實時查詢 Passport
  ├─ 獲取用戶的最新積分
  ├─ 顯示在個人頁面
  └─ 計算特權和折扣
```
