### 2. 郵件通知修復 (2026-03-06)

**根本問題:**
```
RESEND_FROM_EMAIL 在 .env.local 中為空
→ Resend API 無法發送郵件
→ 客戶未收到訂單確認郵件
```

**修復方案:**
```
/.env.local
  修改: RESEND_FROM_EMAIL=noreply@shop.kiwimu.com
  
驗證:
  ✓ /lib/notifications.ts - sendCustomerEmail() 已正確實現
  ✓ /app/api/order/route.ts - 郵件發送已觸發
  ✓ 郵件範本包含會員中心連結
```

**郵件流程:**
```
POST /api/order
  ↓
createOrder() 成功
  ↓
Promise.all([
  sendCustomerEmail() → Resend API ✅
  sendDiscordNotify() → Discord Webhook
  notifyNewOrder() → LINE/通知系統
])
```

## 🔍 故障排查指南

### 問題 1: 會員中心頁面加載失敗
```
排查步驟:
1. 確認 Supabase 連接正常
   - 檢查 NEXT_PUBLIC_SUPABASE_URL
   - 驗證 NEXT_PUBLIC_SUPABASE_ANON_KEY

2. 檢查認證狀態
   - 打開瀏覽器開發工具 F12
   - Console 檢查是否有認證錯誤
   - Network 檢查 /api/user/orders 響應

3. 檢查 RLS 政策
   - Supabase Dashboard → Policies
   - 驗證 users/orders 表政策正確

解決方案:
  如是新環境，可能需要更新 RLS 政策
  參考: /docs/ADMIN_DEPLOYMENT_GUIDE.md
```

### 問題 2: 郵件未發送
```
排查步驟:
1. 檢查環境變數
   cat .env.local | grep RESEND
   確保 RESEND_FROM_EMAIL 已設置

2. 檢查 Resend API 額度
   登入 Resend 控制台檢查使用情況
   免費版每月 100 封郵件

3. 檢查郵件垃圾文件夾
   郵件可能被標記為垃圾
   添加 noreply@shop.kiwimu.com 到聯絡人

4. 檢查日誌
   npm run dev 控制台輸出
   檢查 sendCustomerEmail() 是否調用

解決方案:
  更新 .env.local 並重啟 npm run dev
  或升級 Resend 付費方案 (如需大量發送)
```

### 問題 3: 訂單無法與用戶關聯
```
排查步驟:
1. 驗證用戶已登入
   檢查 localStorage 中是否有 Supabase session

2. 檢查 /app/api/order/route.ts 修改
   確保包含:
   - import { createClient } from '@/lib/supabase-server'
   - supabaseServer.auth.getUser()
   - user_id: user_id || currentUser?.id || null

3. 檢查數據庫
   Supabase 查看 orders 表
   新訂單是否包含 user_id

解決方案:
  重新構建項目: npm run build
  或檢查是否遺漏代碼修改
```
